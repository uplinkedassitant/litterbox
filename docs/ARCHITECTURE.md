## Current Test Error Explained

The test suite shows 2 failing tests with this error:
```
Error: AnchorError caused by account: current_cycle. 
Error Code: ConstraintSeeds. 
Error Number: 2006. 
Error Message: A seeds constraint was violated.
```

### Root Cause
The failing tests are in `full-flow-test.ts` which tests the **deposit** functionality. The issue is:

1. **Test is using Cycle 0** (the genesis cycle created at initialization)
2. **Program expects Cycle 1** (after launch, `config.current_cycle = 1`)
3. **PDA derivation mismatch** - the test derives PDA for cycle 0, but program expects cycle 1

### Why This Happens
After the `launch()` instruction:
- `config.current_cycle` is set to `1`
- The `deposit()` instruction uses `config.current_cycle` to derive the cycle PDA
- Test is still pointing to cycle 0's PDA

## Program Architecture

### Key PDAs
```typescript
// Config (global state)
const [configPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("config")],
  programId
);

// Cycles (one per cycle ID)
const [cycle0Pda] = PublicKey.findProgramAddressSync(
  [Buffer.from("cycle"), new BN(0).toArrayLike(Buffer, "le", 8)],
  programId
);

const [cycle1Pda] = PublicKey.findProgramAddressSync(
  [Buffer.from("cycle"), new BN(1).toArrayLike(Buffer, "le", 8)],
  programId
);

// Platform Token Mint
const [platformMintPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("platform_token")],
  programId
);
```

### State Machine
```
1. Initialize ŌåÆ Creates Config + Cycle 0
   - config.launched = false
   - config.current_cycle = 0

2. Launch ŌåÆ Creates Platform Token + Cycle 1
   - config.launched = true
   - config.current_cycle = 1  ŌåÉ Key change!
   - Cycle 1 PDA created

3. Deposit ŌåÆ Uses config.current_cycle to find active cycle
   - Derives PDA: [b"cycle", &config.current_cycle.to_le_bytes()]
   - After launch, this = Cycle 1
```

## Frontend Integration Points

### 1. Initialize Platform
```typescript
await program.methods
  .initialize()
  .accounts({
    config: configPda,
    firstCycle: cycle0Pda,  // ŌåÉ Cycle 0
    feeVault: feeVaultAddress,
    authority: userPublicKey,
    systemProgram: SystemProgram.programId,
  })
  .rpc();
```

### 2. Launch Platform
```typescript
await program.methods
  .launch()
  .accounts({
    config: configPda,
    platformTokenMint: platformMintPda,
    airdropVault: airdropVaultAddress,
    cycle1: cycle1Pda,  // ŌåÉ Cycle 1 (created atomically)
    authority: userPublicKey,
    tokenProgram: TOKEN_PROGRAM_ID,
    associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
    systemProgram: SystemProgram.programId,
  })
  .rpc();
```

### 3. Deposit Tokens
```typescript
// Get current cycle from config
const config = await program.account.config.fetch(configPda);
const currentCycleId = config.currentCycle.toNumber();

// Derive PDA for current cycle
const [currentCyclePda] = PublicKey.findProgramAddressSync(
  [Buffer.from("cycle"), new BN(currentCycleId).toArrayLike(Buffer, "le", 8)],
  programId
);

// Then deposit
await program.methods
  .deposit(amount)
  .accounts({
    config: configPda,
    currentCycle: currentCyclePda,  // ŌåÉ Must match config.current_cycle!
    contributor: contributorPda,
    depositTokenMint: tokenMint,
    userTokenAccount: userTokenAccount,
    tokenVault: tokenVaultPda,
    feeVaultTokenAccount: feeVaultPda,
    feeVaultAuthority: feeVaultAuthorityPda,
    authority: userPublicKey,
    tokenProgram: TOKEN_PROGRAM_ID,
    associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
    systemProgram: SystemProgram.programId,
  })
  .rpc();
```
